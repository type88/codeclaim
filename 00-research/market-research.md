# Market Research

## Executive Summary

The promo code distribution market is fragmented and underserved. While there are large coupon management platforms serving retailers (Vouchery.io, Talon.One), **there's no dedicated solution for developers distributing app store promo codes**. This represents a clear market opportunity with minimal direct competition.

**Key findings:**
- Adjacent coupon/promo management market is $30.8B and growing at 26.1% CAGR
- Apple's App Store Connect allows up to 100 promo codes per app version, but distribution is manual
- Developers currently use spreadsheets, notes apps, or basic tools like Dev Promo Kit
- No platform automates platform-specific code distribution (iOS vs Android vs Steam, etc.)

**Recommendation:** Build the MVP immediately. First-mover advantage is real and achievable.

## Market Size & Opportunity

### Primary Market: App Developer Promo Code Distribution
- **Addressable market:** 6.8M mobile app developers worldwide
- **Serviceable market:** ~680K developers who regularly launch new features/apps requiring promo codes
- **Initial target:** 10K developers in first year (conservative 1.5% penetration)

### Adjacent Markets (Secondary Validation)
- **Coupon management platforms:** $30.8B market, 26.1% CAGR through 2033
- **Low-code development platforms:** $101.68B by 2030, 22.5% CAGR
- **AI code tools:** $23.97B by 2030, 26.6% CAGR

**What this means:** Large, growing adjacent markets validate demand for developer tools and automated code management.

## Competitive Landscape

### Direct Competitors: None
**Zero platforms solve the core problem:** Bulk upload promo codes → automatic distribution based on user's platform → real-time tracking.

### Indirect Competitors
1. **Dev Promo Kit** (iOS app)
   - Basic code organization and sharing
   - Manual distribution only
   - $0 revenue model (tips-based)
   - **Gap:** No automation, no platform detection

2. **Apple App Store Connect**
   - Native promo code generation
   - Manual distribution via email/text
   - **Gap:** No bulk management, no cross-platform support

3. **Spreadsheets/Notes apps**
   - Current default solution for most developers
   - **Gap:** Everything—this is the problem we're solving

### Adjacent Players (Not Competing)
- **Vouchery.io:** Enterprise coupon management ($179+/month)
- **Talon.One:** Retail promotion engine (enterprise pricing)
- **Open Loyalty:** Retail-focused coupon software

**Key insight:** All existing solutions serve retailers/ecommerce, not app developers. Different use case entirely.

## Target User Research

### Primary Persona: Indie/Small Team App Developers
**Demographics:**
- 1-5 person development teams
- Launching 2-4 app updates/year requiring promotion
- Revenue: $10K-$500K annually per app
- Platforms: iOS + Android (most common), some add Steam/console

**Pain Points (Validated):**
1. **Manual code distribution:** Copy-pasting codes into emails/DMs
2. **Platform confusion:** Users grab wrong codes (iOS codes for Android users)
3. **Tracking chaos:** No visibility into which codes are used
4. **Time waste:** 2-3 hours per launch managing code distribution
5. **Code expiration:** 28-day Apple limit causes unused codes to expire

**Current Behavior:**
- Generate codes in App Store Connect/Google Play
- Store in spreadsheets or notes
- Manually send to influencers/reviewers/press
- No systematic tracking

### Decision Factors
**Must-haves:**
- Dead simple bulk upload (CSV/paste)
- Automatic platform detection
- Public sharing URLs
- Usage tracking dashboard

**Nice-to-haves:**
- API for integration
- Custom branding
- Analytics/insights
- Team collaboration

**Market timing:** Excellent. App ecosystem is growing, but distribution tools haven't evolved.

## Risk Assessment

### Low Risks
- **Market exists:** Developers clearly have this pain point
- **Simple solution:** Core functionality is straightforward
- **Network effects:** Users will share links, driving organic growth

### Medium Risks
- **Platform dependency:** Relies on App Store Connect, Google Play policies
- **Seasonal usage:** May spike around app launches, quiet between

### High Risks
- **Apple/Google builds competing feature:** Could integrate into native platforms
- **Mitigation:** First-mover advantage, superior UX, multi-platform support
